## 2. 추가로 구현해야 할 기능들

### 2.1 실시간 알림 시스템
**우선순위: 높음**

- [ ] 읽지 않은 메시지 수 표시 (대화방 목록, 헤더)
- [ ] 읽지 않은 DM 수 표시 (크리에이터 인박스)
- [ ] 새 후원 알림 (크리에이터)
- [ ] 실시간 메시지 업데이트 (WebSocket 또는 폴링)
- [ ] 브라우저 알림 (선택적)

**구현 방법:**
- React Query의 `refetchInterval` 활용 (현재 5초)
- 또는 WebSocket 연결로 실시간 업데이트
- 읽지 않은 메시지 수는 API 응답에 포함

### 2.2 후원 완료 후 자동 메시지 등록
**우선순위: 높음**

- [ ] 후원 완료 페이지에서 메시지 자동 등록
- [ ] Intent ID와 메시지 연결
- [ ] 후원 상태 확인 (OCCURRED → CONFIRMED)

**현재 상태:**

## 3. API 엔드포인트 추가 필요

### 3.1 읽지 않은 메시지 수
```
GET /conversations/unread-count
GET /creator/inbox/unread-count
```

### 3.2 후원 완료 후 메시지 등록
```
POST /donations/{intentId}/complete
Body: { message: string }
```

### 3.3 크리에이터 통계
```
GET /creator/stats
Query: ?period=day|week|month
```

### 3.4 프로필 업데이트
```
PUT /profile
Body: { display_name?: string, profile_image?: string }
```

## 4. 데이터베이스 스키마 추가 필요

### 4.1 알림 테이블
```sql
CREATE TABLE notifications (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  type VARCHAR(50), -- 'message', 'donation', 'follow'
  content TEXT,
  read BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### 4.2 메시지 읽음 상태
```sql
ALTER TABLE messages ADD COLUMN read_by_creator BOOLEAN DEFAULT false;
ALTER TABLE messages ADD COLUMN read_at TIMESTAMP;
```
